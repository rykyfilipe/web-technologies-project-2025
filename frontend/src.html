Specificație a Cerințelor Software (SRS)

Autor: Eduard Ozarchevici,Bondor Ricardo Filipe

Proiect: Aplicație Web - Nominalizări filme, actori și voturi

1. Introducere

Acest document descrie cerințele software pentru aplicația web dedicată gestionării nominalizărilor de filme și actori. Aplicația permite utilizatorilor să consulte informații despre filme și actori, să vizualizeze nomin. Este compusă din două componente: un frontend (client) și un backend REST API.

2. Cerințe principale

2.1 Funcționalități esențiale

  • Autentificare și înregistrare utilizatori (JWT token).
  • Vizualizare listă actori, filme și detalii individuale.
  • Vizualizare nominalizări active (ex: pentru Oscar, Globuri de Aur etc.).
  • Vot pentru actori și filme din partea utilizatorilor logați.
  • Adăugare/editare/ștergere filme, actori, nominalizări (doar pentru admin).
  • Filtrare, sortare și căutare în listă filme/actori.

2.2 Interacțiunea cu utilizatorul

Interfața utilizator oferă o experiență prietenoasă și responsive. Utilizatorul accesează aplicația prin browser și are acces la un dashboard cu filme, actori și nominalizări. După autentificare, poate vota sau nominaliza, iar administratorii pot adăuga sau edita informații. Toate acțiunile oferă feedback vizual clar (confirmări, erori, avertismente).

3. Cerințe de interfațare externă

3.1 Interfața cu utilizatorul

Frontend-ul este realizat cu HTML, CSS și JavaScript, fiind livrat dintr-un server separat. Interacțiunea se realizează prin apeluri AJAX către API-ul backend. Aplicația este compatibilă cu dispozitive mobile și desktop, urmând principiile de design responsive.

3.2 Interfața software (API)

API-ul este de tip REST și oferă următoarele endpoint-uri principale:

  • POST /auth/register – înregistrare utilizator
  • POST /auth/login – autentificare și obținere JWT
  • GET /movies – listă filme
  • GET /actors – listă actori
  • GET /nominations – listă nominalizări
  • POST /votes – trimitere vot pentru actor/film
  • POST /movies – adăugare film (admin)
  • PUT /actors/:id – modificare actor (admin)
  • DELETE /nominations/:id – ștergere nominalizare (admin)

API-ul returnează date în format JSON și necesită autentificare prin token JWT pentru acțiunile protejate.

4. Cerințe non-funcționale

  • Performanță: Răspunsurile API-ului trebuie să fie livrate în max. 1s pentru 90% dintre cereri.
  • Securitate: Autentificare bazată pe JWT, protecție la SQL Injection (prepared statements), CORS configurat corect.
  • Disponibilitate: Serverul trebuie să funcționeze 24/7 și să răspundă la cereri concurente.
  • Scalabilitate: Structura permite adăugarea de noi tipuri de conținut (ex: regizori, categorii de premii etc.).
